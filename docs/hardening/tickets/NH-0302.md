## Problem

NH-0202 verified value fidelity through the DIRECT write path (InsertAsync / prepared inserts /
parameter binders). The BULK paths - BulkInsertAsync, bulk update/delete (ExecuteUpdate /
ExecuteDelete), and the provider staging routes - bind values through different code
(EntityDataReader, DataTable staging, COPY writers, multi-row VALUES batching), and history shows
they diverge: SQLite bulk paths once bypassed value converters (silent corruption, fixed and pinned
in the bulk-converter campaign), and MySQL bulk staging silently ROUNDED sub-second temporal values
through DATETIME(0) staging tables (live-proven, fixed). This ticket is the granular bulk-path
value-fidelity matrix on SQLite, worked one cell at a time with the same differential round-trip
method as NH-0202: bulk-write crafted values per type family, dump storage, read back, compare to
what was written, and compare against the direct-path storage form (bulk and direct must store
IDENTICALLY). Provider-specific staging cells (MySQL DATETIME(6), SqlServer staging types) are noted
for live-provider runs and are NOT closable from SQLite-only evidence.
