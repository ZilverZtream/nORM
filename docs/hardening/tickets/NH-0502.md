## Problem

nORM's provider mobility rests on the DatabaseProvider hook system: every provider-specific SQL
divergence must flow through a virtual hook, and each provider's override SET is the executable
statement of where it diverges from the portable default. Nothing currently pins that override set -
a new hook added without per-provider consideration, or an override added/removed casually, changes
cross-provider semantics silently. This ticket builds the per-cell provider-mobility matrix:
an exhaustive hook-override matrix contract (SQLite-closable), DDL type-mapping parity, DML shape
parity, and live-pending behavioural probes for the divergences recorded by earlier matrices
(SqlServer uniqueidentifier ordering, MySQL native unsigned-64).
