## Problem

The shared query visitor (`ExpressionToSqlVisitor.Members.cs`) branched on
`_provider is SqlServerProvider` in `GetRegexPatternSql`/`GetRegexReplacementSql` to decide
whether a constant `Regex.IsMatch`/`Regex.Replace` argument is emitted as an inline SQL string
literal (SQL Server) or a parameter (others). The behaviour is correct - SQL Server decodes the
pattern literal at translation time (`TryDecodeSqlStringLiteral`) to build a LIKE/PATINDEX shape,
so it needs the literal - but an `is <Provider>` check in the query layer violates the
provider-mobility contract: a provider-specific decision must live on `DatabaseProvider`, not in
shared query code (`feedback_audit_unconditional_provider_syntax`).
