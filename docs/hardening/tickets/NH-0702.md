## Problem

Tenant isolation rests on an expression-rewrite filter (root queries and correlated subqueries),
a fail-closed RequireTenantColumn contract, write-path validation, tenant-scoped includes/M2M/
split-query child loads, temporal trigger tenant guards, and tenant-aware plan/result cache keys.
Substantial adversarial coverage exists (AdversarialTenantFuzzTests, AdversarialTenantObjectMatrix,
AdversarialPerfAndTenantIsolation: fast-path bypass, compiled-plan isolation, count-cache scoping,
tenant-id object matrix). What no suite pins per-cell: the INTERPLAY corners - AsOf temporal reads
against the history table, Cacheable result reuse across tenants, bulk CUD (ExecuteUpdate/Delete)
scoping, boundary-valued tenant ids across query shapes, and the fast-path bail-out as an
executable CONTRACT rather than an implementation accident. This matrix closes those cells with
the established differential method (two tenants, shared table, boundary data; assert BOTH
no-cross-tenant-leak AND no-own-row-loss).
