## Problem

NH-0901 closed the retry-WRITE invariants (never past commit-attempted, db-generated key reset -
`SaveChangesFaultInjectionAtomicity`) and the OCC matrix. What no suite pins per-cell: the
INTERPLAY of the retry strategy with the read pipeline and the result cache
(`RetryingExecutionStrategy` wraps `ExecuteInternalAsync` INCLUDING the cached path - a transient
failure inside the cache factory must retry cleanly, never double-Set or serve a partial result),
read-retry idempotence, fail-loud connection-drop contracts on reads, and per-plan timeout
honoring. This matrix closes those cells with fault-injection probes.

- [ ] RETRY x RESULT CACHE -> with a RetryPolicy configured, a Cacheable read whose FIRST attempt
      fails transiently must succeed on retry with correct data, populate the cache exactly once,
      and later hits serve the correct entry; a transient failure must never cache a partial or
      empty result. PROBE with a fault-injecting command interceptor.
- [ ] READ RETRY IDEMPOTENCE -> a transient failure mid-read retries to a correct, non-duplicated
      result set (tracked entities not double-tracked; aggregates correct).

      surfaces a loud provider exception, never a silent partial list.
- [ ] TIMEOUT CONTRACTS -> plan CommandTimeout / options BaseTimeout reach the command for reads,
      CUD, and bulk paths (pin the wiring; a sub-second-timeout long query throws timeout rather
      than hanging - SQLite-closable via interceptor observation of CommandTimeout).

      (never retry past commit-attempted; db-generated keys reset on retried SaveChanges) - cited,
